Implementation of a Binary Logistic Regression Classification model using numpy¶

Classification is another major task in supervised machine learning, with the objective of predicting the probability of occurence of a particular outcome (the response is usually a categorical variable). Unlike continuous variables, categorical variables are discrete in nature and usually represent an outcome. For example in classification of tumor as malignment or benign, from xray images, the response variable just has 2 values . This is an example of a binary classification problem. The class labels cannot be directly predicted, but a probability of the occurance of a particular label (outcome) can be predicted using an appropriate training algorithm and cost functon. So in the case of the tumor classification, if the predicted probability of the tumor being malignant is 0.7 then using a threshold value (>=0.5), the tumor can be classified as malginant. Threshold values can be set to anything between 0 and 1 but usually 0.5 is used as the class boundary value.

Logistic regression is a very popular method used for binary classification task. The below sections describe how the sigmoid function is used for making predictions in binary classification problems, followed by the definition of cost function used in logistic regression model and the implementation of the training algorithm. Gradient descent is used for the minimizing the cost function in the training process. Metrics like accuracy and confusion matrix are implemented in the module. More metrics can be derived from the confusion matrix (precision, recall etc.) The algorithm is tested on the loan Default dataset from An Introduction to Statistical Learning with Applications in R textbook (http://faculty.marshall.usc.edu/gareth-james/ISL/). Comparison is also made with scikit-learn's LogisticRegression function.

Logistic Regression¶

The logistic regression model revolves around the sigmoid function which is defined as follows, $$ \sigma(Z) = \frac {1}{1+e^{-Z}}$$

Sigmoid varies between 0 and 1, and is plotted below.This property of the sigmoid function makes it a natural choice to predict probabilties of outcomes. The plot also shows the non-linearity of the sigmoid function. This plays a role in the type of the cost function selected in the training process.

The variable Z in the sigmoid function usually represents a summation of terms involving features and coefficients. Generally Z is represented as follows,

$$Z=\beta^{T}X$$

where $\beta$ is the coefficient vector,

$$\beta = \begin {bmatrix} \beta_{0} \\ \beta_{1} \\ . \\ . \\ . \\ \beta_{n} \\ \end{bmatrix}$$

and $X$ is the feature vector $$X=\begin{bmatrix} x_{0} \\ x_{1} \\ . \\ . \\ . \\ x_{n} \\ \end{bmatrix}$$

Here $x_{0} = 1$ and $\beta_{0}$ is the intercept.

Training algorithm¶

The values of $\beta$'s which fit the input data the best are determined by an optimization process. A cost function needs to be defined and the values of $\beta$ which minimize the cost function represent the best fit. What this means is the probability predicted by plugging in the $\beta$ coefficients in the sigmoid function results values close to the class labels. In the case of binary classification the class labels are usually 0 and 1. The cross entropy function defined as follows is used as the loss and cost functions in the optimization process.

$$ Loss_i = - {y_i . \log(\sigma(Z_i)) + (1-y_i) . \log(1-\sigma(Z_i))} $$

The main inituition behind using the above loss function is to make sure the value of $sigmoid(Z)$ is close to y which the actual class label of 0 or 1.

  1. When y = 1, the 2nd term in the loss function is zero. For the loss function to be as small as possible $sigmoid(Z)$ term need to be close to 1.
  2. Similarily when the class label y = 0, the first term in the loss function is zero. For the loss function to be as small as possible $sigmoid(Z)$ value should be close to 0.

Hence the above defintion of the loss function guides the optimization algorithm in the correct directon in both the cases. Cost function which represents the average of the loss function over m training samples is given as below.

$$ Cost J = -\frac{1}{m}\sum_{i=1}^{m}\{y_i . \log(\sigma(Z_i)) + (1-y_i) . \log(1-\sigma(Z_i))\}$$

In this notebbook, a simple gradient descent algorithm is used for minimizing the cost function. The implementation of gradient descent algorithm is as follows.

  1. The $\beta$ coefficient vector is initialized either to random values or just ones.
  2. The gradient of the cost function with respect to parameter coefficient $\beta$ is given as follows
$$ \frac {\partial J}{\partial \beta_0} = \frac{1}{m} \sum_{i=1}^{m} \{y_i - \sigma(\beta^T.X)\}x_0$$$$ \frac {\partial J}{\partial \beta_1} = \frac{1}{m} \sum_{i=1}^{m} \{y_i - \sigma(\beta^T.X)\}x_1$$$$ \frac {\partial J}{\partial \beta_2} = \frac{1}{m} \sum_{i=1}^{m} \{y_i - \sigma(\beta^T.X)\}x_2$$$$.$$$$.$$$$ \frac {\partial J}{\partial \beta_n} = \frac{1}{m} \sum_{i=1}^{m} \{y_i - \sigma(\beta^T.X)\}x_n$$
  1. Once the gradients have been calculated, the new values of $\beta$ which reduces the cost function can be iterated as follows.
$$\beta_j^{new} = \beta_j^{old} - \alpha \frac {\partial J}{\partial \beta_j^{old}}$$

Here $\alpha$ represents the training rate, which can be set to an appropriate value based on convergence of the cost function in the iterative process. The process of fitting the logistic regression model has been captured in the below implementation as a python class.The confusion matrix and model accuracy are available as output from the model. The feature matrix is automatically scaled using a minimum maximum scaler function. The scaler function is similar to MinMaxScaler funtion part of scikit-learn package. This makes the class self contained without any outside library dependence except for numpy.

The confusion matrix and model accuracy scores are also available as output. Please refer to the wikipedia article for the explanation about confusion matrix, precision, recall, f1 score, sensitivity, specificity. https://en.wikipedia.org/wiki/Confusion_matrix. Note the confusion matrix corresponds to a binary classification problem.

class logregmodel():
    def __init__(self):
        self.X=0
        self.y = 0
        
    def sigmoid(self,x):
        return 1/(1+np.exp(-x))
    
    def scaler_fit(self,X):
        self.X_max = X.max(0)
        self.X_min = X.min(0)
            
    def scaler_transform(self,X):
        return (X-self.X_min)/(self.X_max-self.X_min)
    
    def fit(self,X,y,alpha=1e-3,epoch=200):
        '''
        The fit function implements gradient descent algorithm in fitting the coefficients of
        the logistic regression model. The training set X, the training labels y in the form of 0 and 1,
        the learning rate and number of epochs (iterations) are needed as input.
        '''
        #self.scaler = MinMaxScaler().fit(X)
        self.scaler_fit(X)
        X_scaled = self.scaler_transform(X)
        self.X = np.insert(X_scaled,0,1,axis=1)
        self.y = y
        m = (self.X).shape[0]
        n = (self.X).shape[1]
        self.m = m
        self.n  = n
        theta = np.ones(shape=(n,1))
        theta_t = np.transpose(theta)
        X_t = np.transpose(self.X)
        y_t =self.y.reshape(1,-1)
        cost_list = []
        for i in range(1,epoch):
            y_pred = self.sigmoid(np.dot(theta_t,X_t))
            error = y_pred - y_t
            ### binary cross entropy cost function
            cost = -np.sum(y_t*np.log(y_pred)+(1-y_t)*np.log(1-y_pred))/m
            cost_list.append(cost)
            theta_t = theta_t - alpha*np.dot(error,self.X)/m
        self.coefficients_ = theta_t
        self.cost = cost_list
        self.y_pred = self.sigmoid(np.dot(theta_t,X_t))
        
    def predict(self,X_new):
        #X_new = self.scaler.transform(X_new)
        X_new = self.scaler_transform(X_new)
        X_new = np.insert(X_new,0,1,axis=1)
        m = X_new.shape[0]
        X_t = np.transpose(X_new)        
        y_class = np.zeros(m)
        y_pred = self.sigmoid(np.dot(self.coefficients_,X_t))
        y_pred = y_pred.reshape(-1)
        y_class[y_pred>0.5]=1
        return y_pred,y_class
    
    def confusion_matrix(self,X_new,y_new):
        y_pred,y_class = self.predict(X_new)
        y_mat = np.append(y_new.reshape(-1,1),y_class.reshape(-1,1),axis=1)
        TP = np.sum((y_mat[:,0]==0) & (y_mat[:,1]==0))
        FN = np.sum((y_mat[:,0]==0) & (y_mat[:,1]==1))
        FP = np.sum((y_mat[:,0]==1) & (y_mat[:,1]==0))
        TN = np.sum((y_mat[:,0]==1) & (y_mat[:,1]==1))
        cf = np.zeros(shape=(2,2))
        cf[0,0] = TP
        cf[0,1] = FN
        cf[1,0] = FP
        cf[1,1] = TN
        return cf
        
    def scores(self,X_new,y_new):
        y_pred,y_class = self.predict(X_new)
        accuracy = np.sum(y_class==y_new)/y_new.shape[0]
        return accuracy

Conclusion¶

The logistic regression model was successfully implemented with gradient descent algorithm and just using numpy arrays. The implemented model was compared with scikit-learn's LogisticRegression() function and was found to give essentially he same result. The logistic regression model performs poorly in predicting the loan default rates as the classes are highly imbalanced. The recommendation is to use a slightly modified cost function or use a regularization technique with a penality term to penalize the cost function for predicting defaulted records as not defaulted.
